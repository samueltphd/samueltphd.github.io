Research Interests

  • Secure Memory

    A property of increasingly dense memory devices is that they are increasingly susceptible to corruption based attacks, with limited access to the device and without access permissions. As a result, devices can implement secure memory, a protocol that implements a Bonsai Merkle Tree and uses it to authenticate that values aren't corrupted between when they are stored in memory and the next time they are loaded. Unfortunately, this protocol is too slow to be useful in practice. Emerging memory technologies and use cases provide interesting opportunities and challenges for this protocol.

  • Computational Memory Devices

    Near-memory processing, near-memory compute, processing-in memory, processing-using memory, computational storage devices, etc are all emerging trends in memory that promise high memory bandwidth. Yet, despite the opporunities of co-design applications with the device, their utility may be limited due their value in terms of breadth of use cases, programming model (i.e., coherence), and their generality comes at the cost of implementing on bare metal. I believe there are still interesting opportunities with these technologies, and that their appeal is grounded in legitimate observations from classical computing architectures.

  • Hardware-Aware Software and Concurrent Algorithms

    Persistent memories and non-uniform memories have provided difficult challenges in terms of concurrent data structure access patters. For example, without explicitly persisting values, a concurrent data structure will remain crash inconsistent. Furthermore, naive placement of data will lead to suboptimal performance for applications using far memory. If these applications are made architecture-aware, however, there can also be significant improvements.
